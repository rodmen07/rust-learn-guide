# Capstone Project

Build a small command-line app that uses the ideas from this guide. This is your chance to combine everything into one useful project.

One example is the `todo_cli` crate in `examples/`, which demonstrates structs, vectors, and command-line input.

Suggested features:

- read user input
- store data in a vector
- use structs and enums
- handle errors with `Result`
- add tests

## Project Plan

1. Pick a problem you actually want to solve.
2. Write down the data you need to store.
3. Sketch the commands or actions first.
4. Implement the simplest working version.
5. Add persistence, validation, and tests.

## Example Path

You can use `todo_cli` as a reference for a small but complete app:

- parse command-line input
- load and save tasks
- mark items as done
- print a readable task list

## Finish Strong

- keep the scope small
- commit one feature at a time
- write tests for the trickiest logic
- stop when the app is useful, not perfect

## Exercise

1. Pick a small problem to solve.
2. Design the data types first.
3. Implement it one piece at a time.

## Challenge

1. Add one more feature to your app.
2. Save the data to a file.
3. Share the project with someone else.
